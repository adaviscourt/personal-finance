# Product

## Register

product

## Users

Primary user is Austin managing personal finances locally. The app should stay simple enough for personal use while remaining understandable for other people who may run it on their own machine later.

## Product Purpose

Personal Finance helps review, import, label, and understand transactions from local bank CSVs. Success means Austin can open the app, see the current month's transaction activity clearly, filter by month/account/label, and use separate guided workflows for import and labeling without the dashboard feeling like a development console.

## Brand Personality

Clean, trustworthy, guided.

The interface should borrow from FreeTaxUSA's straightforward confidence: clear workflow steps, restrained visual styling, friendly green/blue branding, and plain language that makes data tasks feel safe and manageable.

## Anti-references

Avoid the current crowded home page where dashboard, account management, import, templates, preview, and labeling all compete at once.

Avoid developer-facing status UI like health checks on user-facing pages.

Avoid pastel panel sprawl, oversized hero treatment, decorative section cards, and chart-first dashboard layouts that obscure transaction review.

## Design Principles

Dashboard first: the home page should focus on monthly transaction review and filtering.

Workflows belong in modules: importing, labeling, and account management should live in dedicated areas with clear entry points.

Trust through clarity: use familiar tables, labels, controls, and validation instead of decorative UI patterns.

Guidance without ceremony: complex tasks like importing should feel step-by-step, but not like a heavy wizard unless the task needs it.

Reference FreeTaxUSA carefully: use its trustworthy simplicity, guided flow, and green/blue brand cues while adapting them to a dense product dashboard.

## Accessibility & Inclusion

Target WCAG AA for contrast, focus states, forms, and table readability. Support keyboard navigation across filters, tables, and import workflows. Respect reduced-motion preferences. Do not rely on color alone for statuses, labels, or transaction direction.
